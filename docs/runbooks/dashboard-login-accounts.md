# Dashboard Login Accounts

The dashboard login is independent from Finale portal automation. QA users should sign in directly with dashboard accounts. Playwright only uses the portal credentials in the backend intake service to collect workbook, OASIS, referral, and billing-period evidence.

## Account Model

Dashboard accounts are configured with `DASHBOARD_QA_USERS_JSON`.

Each account has:

- `email`: login email for the QA user.
- `name`: display name.
- `passwordHash`: SHA-256 password hash generated by the dashboard helper.
- `allowedAgencyIds`: agency queues the user can open.

Supported agency ids:

- `aplus-home-health`
- `active-home-health`
- `avery-home-health`
- `meadows-home-health`
- `star-home-health`

## Generate A User

From the repo root:

```powershell
cmd /c pnpm dashboard:qa-user -- --email qa.user@example.com --name "QA User" --agencies active-home-health,star-home-health
```

If `--password` is omitted, the script generates a temporary password and prints it once. Share that password through a secure channel. Store only the printed `passwordHash` in deployment configuration.

To provide a known temporary password:

```powershell
cmd /c pnpm dashboard:qa-user -- --email qa.user@example.com --name "QA User" --password "temporary-long-password" --agencies active-home-health
```

## Configure Multiple Users

Combine generated user objects into one JSON array:

```json
[
  {
    "email": "qa.one@example.com",
    "name": "QA One",
    "passwordHash": "sha256:...",
    "allowedAgencyIds": ["active-home-health", "star-home-health"]
  },
  {
    "email": "qa.two@example.com",
    "name": "QA Two",
    "passwordHash": "sha256:...",
    "allowedAgencyIds": ["aplus-home-health", "avery-home-health", "meadows-home-health"]
  }
]
```

Set this full array as `DASHBOARD_QA_USERS_JSON`.

## Production Rules

- Set `DASHBOARD_SESSION_SECRET` to a unique secret with at least 32 characters.
- Set `DASHBOARD_ALLOW_PLAINTEXT_PASSWORDS=false`.
- Use `passwordHash`, not plaintext `password`, for every production user.
- Store `DASHBOARD_QA_USERS_JSON` and `DASHBOARD_SESSION_SECRET` in AWS Secrets Manager or an equivalent secret store.
- Updating a password means generating a new `passwordHash`, updating the secret value, and redeploying/restarting the dashboard task.

## Login Flow

1. User opens `/login`.
2. User enters dashboard email/password.
3. The dashboard verifies credentials from `DASHBOARD_QA_USERS_JSON`.
4. The dashboard sets a signed session cookie.
5. User chooses one assigned agency.
6. The dashboard opens `/agency` and reads the latest processed API data for that agency.

No QA user needs Finale portal credentials to access the dashboard.
